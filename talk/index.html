what is docker?

"Docker is an open-source engine that automates the deployment of any application as a lightweight, portable, self-sufficient container that will run virtually anywhere."

  • Automate the packaging and deployment of applications
  • Create lightweight, private PAAS environments
  • Automate testing and continuous integration/deployment
  • Deploy and scale web apps, databases and backend services

what is docker not? (yet)

  • fully integrated front-end environment. (edit on OS-X, view in browser)
  • a traditional VM
  • i.e. Windows, OS-X, other non-linux OSs
  • supported by frog IT

huh? (what's the problem)

  • You need node/ruby/python/apache/ngingx for a project
  • You're not alone - other ppl are working on a project
  • "It works for me" but not for QA
  • The client needs to deploy your project to their infrastructure/AWS/rackspace

historical solutions

  • 1998 - buy a server, copy image to machine (Symatec/Norton Ghost)
  • 2000 - create a virtual machine - provision in minutes
  • 2014 - create a container - provision in seconds
  • [type 1 hypervisors, type 2 hypervisor, linux container]

why?

  • one stack to rule them all - hardware, software stack
  • separation of concerns - design principal to separate system into subsystems, each addressing a separate concern
  • modularity
  • run anywhere (anywhere running linux...)
  • hundreds of existing dockerized apps (node, memcached, redid, mongo, hadoop)
  • integrates into CI & VM systems - jenkins, travis, chef, puppet, vagrant, openstack
  • github

How does it work?

  • it uses Linux Containers (LXC)
  • (no need for a hypervisor like VMWare/VirtualBox/XenServer/Hyper-V)
  • cgroups - limits, accounting
  • namespaces - process-based resource isolation (processes, filesystem, networking, uid & gid)
  • chroot - apparent root filesystem access

how to? (managing docker, best practices)

  • docker command-line in linux
  • boot2docker command-line in OS-X
  • Dockerfile for automation
